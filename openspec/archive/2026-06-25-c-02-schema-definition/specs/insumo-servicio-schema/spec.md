## ADDED Requirements

### Requirement: InsumoServicio collection exists in PocketBase
The system SHALL have an `insumoServicio` collection in PocketBase with the following fields and configuration.

#### Fields

| Field | Type | Required | Default | Constraints |
|-------|------|----------|---------|-------------|
| id | text (PK) | auto | — | Auto-generated by PocketBase |
| nombre | text | sí | — | Must be unique? No (not specified in KB) |
| stockActual | number | sí | 0 | Must be >= 0 |
| unidad | text | sí | — | e.g. "unidad", "resma", "cartucho" |
| costoUnitario | number | sí | 0 | Cost per unit in ARS |
| stockMinimo | number | sí | 0 | Alert threshold |
| paginasPorUnidad | number | no | null | Nullable; only for toner/ink supplies |

#### API Rules
- Public CRUD: list, view, create, update, delete allowed without authentication

#### Scenarios

- **Scenario: InsumoServicio collection is queryable**
  - **GIVEN** PocketBase is running
  - **WHEN** sending a GET request to `/api/collections/insumoServicio/records`
  - **THEN** the response returns a paginated list of insumoServicio records

- **Scenario: New insumo can be created with paginasPorUnidad**
  - **GIVEN** PocketBase is running
  - **WHEN** sending a POST request with `{ "nombre": "Tóner HP 123", "stockActual": 2, "unidad": "cartucho", "costoUnitario": 4500, "stockMinimo": 1, "paginasPorUnidad": 3000 }`
  - **THEN** the record is created with `paginasPorUnidad: 3000`

- **Scenario: Insumo can be created without paginasPorUnidad**
  - **GIVEN** PocketBase is running
  - **WHEN** sending a POST request with `{ "nombre": "Resma A4", "stockActual": 10, "unidad": "resma", "costoUnitario": 800, "stockMinimo": 2 }`
  - **THEN** the record is created with `paginasPorUnidad: null`

### Requirement: MovimientoInsumo collection exists in PocketBase
The system SHALL have a `movimientoInsumo` collection in PocketBase with the following fields and configuration.

#### Fields

| Field | Type | Required | Default | Constraints |
|-------|------|----------|---------|-------------|
| id | text (PK) | auto | — | Auto-generated by PocketBase |
| insumoServicio_id | relation → InsumoServicio | sí | — | References `insumoServicio` collection |
| tipo | select | sí | — | One of: "ingreso", "consumo" |
| cantidad | number | sí | — | Must be > 0 |
| fechaHora | autodate | auto | — | Auto-set on create |
| observacion | text | no | — | Optional free-text note |

#### API Rules
- Public CRUD: list, view, create, update, delete allowed without authentication

#### Scenarios

- **Scenario: MovimientoInsumo collection is queryable**
  - **GIVEN** PocketBase is running
  - **WHEN** sending a GET request to `/api/collections/movimientoInsumo/records`
  - **THEN** the response returns records with all defined fields

- **Scenario: Movimiento creates with relation**
  - **GIVEN** an insumoServicio record with `id: "abc123"`
  - **WHEN** sending a POST to `/api/collections/movimientoInsumo/records` with `{ "insumoServicio_id": "abc123", "tipo": "consumo", "cantidad": 2 }`
  - **THEN** the record is created with the relation and auto-set `fechaHora`

### Requirement: Insumos frontend service layer
The system SHALL have a service module at `src/features/insumos/api/insumosService.ts` providing typed async functions for both `insumoServicio` and `movimientoInsumo` operations.

#### Functions
- `listSupplies()` → `Promise<InsumoServicio[]>` — fetch all supplies
- `getSupplyById(id: string)` → `Promise<InsumoServicio>` — fetch single supply
- `createSupply(data: CreateInsumoInput)` → `Promise<InsumoServicio>` — create new supply
- `updateSupply(id: string, data: UpdateInsumoInput)` → `Promise<InsumoServicio>` — update supply
- `deleteSupply(id: string)` → `Promise<boolean>` — delete supply
- `listMovements(supplyId?: string)` → `Promise<MovimientoInsumo[]>` — list movements, optionally filtered by supply
- `recordMovement(data: CreateMovimientoInput)` → `Promise<MovimientoInsumo>` — record ingreso/consumo

#### Scenario: Insumos service lists supplies
- **GIVEN** the insumos service is imported
- **WHEN** calling `insumosService.listSupplies()`
- **THEN** a GET request is sent to `/api/collections/insumoServicio/records` and the results are returned
