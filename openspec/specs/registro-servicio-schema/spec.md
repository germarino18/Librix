## ADDED Requirements

### Requirement: RegistroServicio collection exists in PocketBase
The system SHALL have a `registroServicio` collection in PocketBase with the following fields and configuration.

#### Fields

| Field | Type | Required | Default | Constraints |
|-------|------|----------|---------|-------------|
| id | text (PK) | auto | — | Auto-generated by PocketBase |
| fecha | date | sí | — | ISO date format (YYYY-MM-DD) |
| tipo | select | sí | — | One of: "fotocopia", "plastificado", "souvenir", "otro" |
| descripcion | text | sí | — | Description of the service rendered |
| cantidad | number | sí | — | Quantity (e.g., number of fotocopias) |
| ingresoTotal | number | sí | 0 | Total income from this service in ARS |
| costoInsumos | number | no | 0 | Cost of supplies used (optional, default 0) |
| ganancia | number | sí | 0 | `ingresoTotal - costoInsumos`. Calculated by client **before** write — PocketBase does not support auto-formulas |

#### Important Implementation Note
`ganancia` must be calculated by the frontend service layer (or API caller) before writing the record. PocketBase does NOT support computed/virtual fields. The calculation is: `ganancia = ingresoTotal - costoInsumos`.

#### API Rules
- Public CRUD: list, view, create, update, delete allowed without authentication

#### Scenarios

- **Scenario: RegistroServicio collection is queryable**
  - **GIVEN** PocketBase is running
  - **WHEN** sending a GET request to `/api/collections/registroServicio/records`
  - **THEN** the response returns a paginated list of registroServicio records

- **Scenario: New service record can be created with ganancia**
  - **GIVEN** PocketBase is running
  - **WHEN** sending a POST to `/api/collections/registroServicio/records` with `{ "fecha": "2026-06-24", "tipo": "fotocopia", "descripcion": "Fotocopias A4 b/n", "cantidad": 100, "ingresoTotal": 500, "costoInsumos": 100, "ganancia": 400 }`
  - **THEN** the record is created with `ganancia: 400`

- **Scenario: Create service record without costoInsumos**
  - **GIVEN** PocketBase is running
  - **WHEN** sending a POST with `{ "fecha": "2026-06-24", "tipo": "souvenir", "descripcion": "Llavero personalizado", "cantidad": 1, "ingresoTotal": 1500, "ganancia": 1500 }`
  - **THEN** the record is created with `costoInsumos: 0` and `ganancia: 1500`

### Requirement: RegistroServicio frontend service layer
The system SHALL have a service module at `src/features/servicios/api/serviciosService.ts` providing typed async functions for all RegistroServicio operations.

#### Functions
- `list()` → `Promise<RegistroServicio[]>` — fetch all service records
- `getById(id: string)` → `Promise<RegistroServicio>` — fetch single record
- `create(data: CreateServicioInput)` → `Promise<RegistroServicio>` — create new record (calculates ganancia automatically)
- `update(id: string, data: UpdateServicioInput)` → `Promise<RegistroServicio>` — update record (recalculates ganancia)

#### Important: ganancia auto-calculation
The `create` and `update` functions SHALL auto-calculate `ganancia = ingresoTotal - costoInsumos` before sending to PocketBase, so the caller does not need to compute it manually.

#### Scenario: Servicios service creates record with auto-calculated ganancia
- **GIVEN** the servicios service is imported
- **WHEN** calling `serviciosService.create({ fecha: "2026-06-24", tipo: "fotocopia", descripcion: "Fotocopias", cantidad: 100, ingresoTotal: 500, costoInsumos: 100 })`
- **THEN** the service calculates `ganancia: 400` and sends a POST request with all fields including `ganancia`
