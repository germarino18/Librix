## ADDED Requirements

### Requirement: Caja collection exists in PocketBase
The system SHALL have a `caja` collection in PocketBase with the following fields and configuration.

#### Fields

| Field | Type | Required | Default | Constraints |
|-------|------|----------|---------|-------------|
| id | text (PK) | auto | — | Auto-generated by PocketBase |
| fecha | date | sí | — | ISO date format (YYYY-MM-DD) |
| montoInicial | number | sí | 0 | Must be >= 0 |
| montoFinal | number | no | null | Set when estado changes to "cerrada" |
| estado | select | sí | "abierta" | One of: "abierta", "cerrada" |
| totalEfectivo | number | no | 0 | Sum of cash sales for the day |
| totalTransferencia | number | no | 0 | Sum of transfer sales for the day |
| totalQR | number | no | 0 | Sum of QR sales for the day |
| totalServicios | number | no | 0 | Sum of service income for the day |
| observacion | text | no | — | Optional free-text note |

#### API Rules
- Public CRUD: list, view, create, update, delete allowed without authentication

#### Scenarios

- **Scenario: Caja collection is queryable**
  - **GIVEN** PocketBase is running
  - **WHEN** sending a GET request to `/api/collections/caja/records`
  - **THEN** the response returns a paginated list of caja records with all defined fields

- **Scenario: New caja record can be created**
  - **GIVEN** PocketBase is running
  - **WHEN** sending a POST request to `/api/collections/caja/records` with `{ "fecha": "2026-06-24", "montoInicial": 5000, "estado": "abierta" }`
  - **THEN** the response returns the created record with an auto-generated `id`

- **Scenario: Caja estado can be updated to "cerrada"**
  - **GIVEN** a caja record with `estado: "abierta"`
  - **WHEN** sending a PATCH request to `/api/collections/caja/records/{id}` with `{ "estado": "cerrada", "montoFinal": 15000 }`
  - **THEN** the record is updated and `estado` reflects "cerrada"

### Requirement: Caja frontend service layer
The system SHALL have a service module at `src/features/caja/api/cajaService.ts` providing typed async functions for all Caja CRUD operations.

#### Functions
- `list()` → `Promise<Caja[]>` — fetch all caja records
- `getById(id: string)` → `Promise<Caja>` — fetch single record
- `create(data: CreateCajaInput)` → `Promise<Caja>` — create new caja
- `update(id: string, data: UpdateCajaInput)` → `Promise<Caja>` — update existing caja
- `getTodayCaja()` → `Promise<Caja | null>` — get today's open caja, returns null if none
- `closeCaja(id: string, totals: CloseCajaInput)` → `Promise<Caja>` — close caja with final totals

#### Types
The service SHALL use typed interfaces exported from `src/features/caja/types/index.ts`:

```typescript
interface Caja {
  id: string
  fecha: string
  montoInicial: number
  montoFinal: number | null
  estado: "abierta" | "cerrada"
  totalEfectivo: number
  totalTransferencia: number
  totalQR: number
  totalServicios: number
  observacion: string
}
```

#### Scenario: Caja service creates a record
- **GIVEN** the caja service is imported
- **WHEN** calling `cajaService.create({ fecha: "2026-06-24", montoInicial: 5000, estado: "abierta" })`
- **THEN** a POST request is sent to `/api/collections/caja/records` and the created record is returned
