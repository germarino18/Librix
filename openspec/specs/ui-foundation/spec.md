## ADDED Requirements

### Requirement: Tailwind CSS v4 configured
The project SHALL use Tailwind CSS v4 with the Vite plugin (`@tailwindcss/vite`). Configuration SHALL be done via CSS using `@import "tailwindcss"` and `@theme` — NOT via `tailwind.config.js`.

#### Scenario: Tailwind classes render in browser
- **WHEN** a component uses Tailwind utility classes (e.g., `text-lg`, `bg-blue-500`)
- **THEN** the styles are applied correctly in the browser

#### Scenario: @theme custom values work
- **WHEN** defining custom theme values with `@theme { --color-primary: #... }` in the main CSS
- **THEN** the custom values are available as Tailwind utilities (e.g., `bg-primary`)

### Requirement: shadcn/ui init configured
The project SHALL run `shadcn init` to set up the components configuration. Components SHALL be installed individually with `shadcn add <component>`.

#### Scenario: shadcn/ui Button component renders
- **WHEN** importing `Button` from `@/components/ui/button`
- **THEN** the Button component renders with correct shadcn/ui styling

### Requirement: Base shadcn/ui components installed
The following shadcn/ui components SHALL be installed: `button`, `card`, `dialog`, `input`, `select`, `table`, `form`, `label`, `toast`, `sonner`.

#### Scenario: All base components can be imported
- **WHEN** importing each component from `@/components/ui/<component>`
- **THEN** each import resolves without errors

### Requirement: CSS variables for theming
The main CSS file SHALL use CSS variables generated by shadcn/ui for colors, borders, and other design tokens, ensuring consistent theming across components.

#### Scenario: CSS variables are defined in root
- **WHEN** inspecting `:root` in the main CSS
- **THEN** CSS variables for colors, borders, rings, and radius are defined
